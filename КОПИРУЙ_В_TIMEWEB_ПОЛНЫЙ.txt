==========================================
ПОЛНЫЙ ДЕПЛОЙ НА TIMEWEB: ФРОНТ + БЭКЕНД
==========================================

ШАГ 1: БАЗА ДАННЫХ

1. Откройте Timeweb Cloud → Ваш проект
2. Создайте PostgreSQL базу данных
3. Откройте SQL Editor
4. Скопируйте содержимое TIMEWEB_FULL_SCHEMA.sql
5. Выполните скрипт

ШАГ 2: БЭКЕНД

1. Создайте server/.env из server/env.example
2. Заполните данные БД Timeweb
3. Сгенерируйте JWT_SECRET: openssl rand -base64 32
4. В Timeweb Cloud создайте App Platform приложение
5. Подключите GitHub репозиторий
6. Root Directory: server
7. Build Command: npm install && npm run build
8. Start Command: npm start
9. Port: 3001
10. Добавьте все переменные из server/.env
11. Задеплойте
12. Запишите URL бэкенда

ШАГ 3: ФРОНТЕНД (ВАРИАНТ A - СТАТИЧЕСКИЙ ХОСТИНГ)

1. Создайте .env в корне проекта:
   VITE_API_URL=https://ваш-бэкенд-url.twc1.net
   VITE_WS_URL=wss://ваш-бэкенд-url.twc1.net

2. Соберите фронтенд:
   npm install
   npm run build

3. В Timeweb Cloud создайте статический сайт
4. Загрузите содержимое папки dist/

ШАГ 3: ФРОНТЕНД (ВАРИАНТ B - APP PLATFORM)

1. В Timeweb Cloud создайте новое App Platform приложение
2. Подключите GitHub репозиторий
3. Root Directory: . (корень проекта)
4. Build Command: npm install && npm run build
5. Start Command: npx serve -s dist -l 3000
6. Port: 3000
7. Environment Variables:
   VITE_API_URL=https://ваш-бэкенд-url.twc1.net
   VITE_WS_URL=wss://ваш-бэкенд-url.twc1.net
8. Задеплойте

ШАГ 4: ОБНОВЛЕНИЕ CORS

После получения домена фронтенда:

1. В бэкенд приложении обновите ALLOWED_ORIGINS:
   https://ваш-фронтенд-домен.twc1.net

2. Перезапустите бэкенд

ШАГ 5: ПРОВЕРКА

1. Откройте домен фронтенда
2. Проверьте консоль (F12) - нет ошибок
3. Попробуйте зарегистрироваться/войти
